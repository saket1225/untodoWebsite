Getting Started

untodo is organized around four main tabs: Today, Progress, Wallpaper, and Settings. Everything revolves around your daily task list.

1

Add Your First Task

Tap the add button on the Today tab. Give it a title, optionally set a priority, category, estimated time, or subtasks.

2

Complete Tasks

Swipe right to complete, swipe left to delete, or long-press for quick actions. Watch your daily score climb toward an A+.

3

Track Your Progress

Head to the Progress tab to see your streak, weekly chart, heatmap, and achievements. Set up a dynamic wallpaper to stay motivated.

Silicon Integration

Connect untodo to a Silicon AI instance via Firebase Firestore. This enables remote task management, daily summaries, weekly reviews, wallpaper configuration, and push notification nudges — all controlled by your AI partner.

How to Connect

1

Open Silicon Settings

In the untodo app, go to Settings → Silicon. You'll see your username and a generated 6-character pairing code.

2

Share Credentials

Send your Silicon instance the following message:

Connect to my untodo app.
Username: [your_username]
Pairing code: [your_code]
Docs: https://untodo.netlify.app/docs.html
3

Start Using

Silicon authenticates with Firebase using email/password auth and begins listening to your silicon_commands collection. Commands are processed in real-time.

Architecture

The Silicon bridge uses a command/response pattern via Firebase Firestore:

Silicon
Writes command to Firestore
→
Firebase
Real-time sync
→
untodo App
Processes & responds
For Silicon Instances: Set a cron to check https://untodo.netlify.app/docs.html periodically to stay updated with the latest available commands and features.

Firebase Structure

Silicon communicates with untodo through two Firestore collections, scoped by username:

Firestore Collections 
// Commands (Silicon → App)
users/{username}/silicon_commands/{commandId}
  ├── type: string          // e.g. "add_task"
  ├── payload: object       // command-specific data
  ├── status: string        // "pending" | "processing" | "done"
  └── createdAt: timestamp

// Responses (App → Silicon)
users/{username}/silicon_responses/{commandId}
  ├── result: object        // response data
  ├── status: string        // "success" | "error"
  └── completedAt: timestamp

Command Lifecycle

  1. Silicon writes a document to silicon_commands/{commandId} with status: 'pending'
  2. The untodo app detects the new command via Firestore real-time listener
  3. App sets status to 'processing', executes the command
  4. App writes the result to silicon_responses/{commandId}
  5. App sets command status to 'done'
